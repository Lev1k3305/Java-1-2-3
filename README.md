Лабораторные работы по программированию

Описание проекта

Этот репозиторий содержит три лабораторные работы по программированию на Java, выполненные в рамках курса. Каждая работа представляет собой консольное приложение, реализующее различные игровые механики. Ниже приведено краткое описание каждой лабораторной работы, инструкции по запуску и ссылки на соответствующие файлы.

1. Сражение с драконом (HeroCheck.java)

Описание: Консольное приложение, моделирующее допуск героя к бою с драконом, исход сражения, расчёт очков и присвоение титула на основе характеристик героя (имя, возраст, наличие лицензии на меч, броня, уровень силы). Программа проверяет условия допуска к бою, определяет результат сражения и присваивает титул: «Новичок», «Рыцарь» или «Легендарный герой».

Основные функции:





Ввод характеристик героя.



Проверка допуска к бою (возраст > 18 и лицензия на меч или наличие брони).



Определение исхода боя (победа, ничья, поражение) на основе силы.



Расчёт очков и присвоение титула.



Проверка корректности ввода (сила в диапазоне 1–100).

Файл: HeroCheck.java (или в папке dragon/).

2. Мини-RPG (monsterbattle.java)

Описание: Консольная игра, в которой герой сражается с тремя монстрами в последовательных раундах. Игрок выбирает действия (атаковать, лечиться, убежать), а исход боя зависит от случайных факторов. За победу над монстром герой получает опыт, что позволяет повышать уровень и увеличивать атаку.

Основные функции:





Характеристики героя: здоровье (HP), опыт (XP), уровень (LVL), атака (ATTACK).



Три боя с монстрами возрастающей сложности (здоровье и урон зависят от раунда).



Действия игрока: атака, лечение, побег (50% шанс).



Рандомизация урона, лечения и шанса побега с использованием java.util.Random.



Повышение уровня при накоплении опыта.

Файл: monsterbattle.java (или в папке rpg/).

3. Лабиринт (MazeGame.java)

Описание: Консольная игра, в которой герой перемещается по лабиринту 10x10, чтобы достичь выхода, избегая стен и сражаясь с монстрами. Игрок управляет героем с помощью клавиш w, a, s, d, а бои аналогичны механике «Мини-RPG». Игра завершается при достижении выхода или гибели героя.

Основные функции:





Лабиринт 10x10 с символами: стены ('#'), пустые клетки ('_'), игрок ('P'), монстры ('M'), выход ('E').



Управление движением героя и проверка столкновений со стенами.



Бои с монстрами с действиями: атака, лечение, побег.



Рандомизация с использованием java.util.Random.



Отображение лабиринта и характеристик героя после каждого хода.

Файл: MazeGame.java (или в папке labyrinth/).

Установка и запуск





Требования:





Java Development Kit (JDK) версии 8 или выше.



Терминал или IDE (например, IntelliJ IDEA, Eclipse) для компиляции и запуска.



Скачивание:





Склонируйте репозиторий:

git clone <URL_репозитория>



Или скачайте файлы напрямую.



Компиляция и запуск:





Для каждой лабораторной работы выполните следующие шаги:





Перейдите в соответствующую папку (если файлы организованы по папкам: dragon/, rpg/, labyrinth/).



Скомпилируйте файл:

javac HeroCheck.java  # для Сражения с драконом
javac monsterbattle.java  # для Мини-RPG
javac MazeGame.java  # для Лабиринта



Запустите программу:

java HeroCheck
java monsterbattle
java MazeGame



Следуйте инструкциям в консоли для ввода данных или выбора действий.

Контрольные вопросы

Общие вопросы





Что такое класс и объект?
Класс — шаблон для создания объектов, определяющий их свойства и методы. Объект — экземпляр класса. Например, в HeroCheck.java класс HeroCheck обрабатывает логику боя, а объект создаётся при запуске программы.



Для чего используется Scanner?
Класс Scanner (java.util.Scanner) используется для чтения ввода пользователя из консоли. Применяется во всех трёх работах для ввода данных (например, характеристик героя, выбора действий).



Почему важно закрывать Scanner?
Закрытие Scanner (через scanner.close() или try-with-resources) освобождает системные ресурсы, предотвращая утечки памяти. В monsterbattle.java и MazeGame.java используется try-with-resources для автоматического закрытия.

Вопросы к «Сражению с драконом»





Как работают условные операторы?
Условные операторы (if-else) проверяют условия и выполняют соответствующий код. В HeroCheck.java они используются для проверки допуска к бою и исхода сражения.

Вопросы к «Мини-RPG» и «Лабиринту»





Что такое массив и для чего он используется?
Массив — структура данных для хранения элементов одного типа. В monsterbattle.java и MazeGame.java массив int[] hero хранит характеристики героя (HP, XP, LVL, ATTACK), а в MazeGame.java массив char[][] представляет лабиринт.



Чем отличается цикл for от while?
Цикл for используется для известного числа итераций (например, три боя в monsterbattle.java). Цикл while применяется, когда условие завершения динамическое (например, бой до гибели героя или монстра).



Как работает switch-case?
Оператор switch-case проверяет значение переменной и выполняет соответствующий блок case. Используется в monsterbattle.java и MazeGame.java для обработки выбора действий игрока.



Зачем нужен оператор default в switch-case?
Блок default обрабатывает случаи, когда ни один case не соответствует значению. В monsterbattle.java он выводит сообщение о неверном выборе.
